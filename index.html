QuickQA Plugin – Short documentation

The purpose of the plugin is to support quality assurance, performance and transparency. 

QuickQA consists of two main functionatities: Checking the coordinate reference systems (CRS) of layers against the project's CRS and detecting missing spatial indexes.

The source code of the QuickQA plugin is available on GitHub.

Context

CRS – Being consistent with CRS when processing spatial data has a potentially huge impact on performance and accuarcy of your results. Therefore it can be important to aline the CRS of layers loaded with the CRS of the project. on the fly transformation

Spatial Indexes – When processing data, especially large datasets, without spatial index is likely to have an impact on the time spent when running tools, etc. It is therefore beneficial for the perfomance of your workflow to check whether all layers have a spatial index and, where applicable, create one.

 

Usage

Find the plugin in the Toolbars section and activate it. The toolbar will appear as follows:

 

Let's start with checking CRS. There are three options available to determine which layers to select for checking. Clicking any of the buttons immediately triggers the search.

  1. Check CRS of all layers – Using this option means all layers loaded to the table of content (ToC) will be checked whether their CRS is the same as the one associated with the project.

  2. Check CRS of active layers – Only the layers ticked to be active will be checked.

  3. Check CRS of selected layers – Only the manually selected layers will be checked.

If all CRS of the layers align with the project's CRS, a messagebar will temporarily appear at the top of the map canvas.

 

If one of more layer is found to misalign, a dialog will appear to list the result.

 

The second main button in the toolbar serves the spatial index functionality.

Important: Shapefile and Geopackage are currently the only two data providers supported.

All Shapefile and Geopackage layers in the project will be detected and checked whether a spatial index exists or not. If there is at least one layer found having no index, a dialog comes up listing the result and the option to create the index. If there are no shapefiles and geopackages without spatial index to be found, a message bar will temporarily appear at the top of the map canvas.
